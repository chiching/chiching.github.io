Hub Hardening#

These hardening steps are in order to satisfy the requiment of MTR's server security, more detailed requirments please refer to the Unix Server Security Hardening Guide.pdf

2. Account Policy#

  • edit file /etc/ssh/sshd_config, to make sure following configuration.

    PermitRootLogin no PasswordAuthentication no ChallengeResponseAuthentication no KerberosAuthentication no GSSAPIAuthentication no X11Forwarding no PrintMotd no

  • sudo systemctl restart sshd

2.1 Privileged Accounts#

Requirments: Automatic account lockout after consecutive login failures is not implemented because it can be used as a form of denial of service. It may lock out the entire server if the root account is locked.

Action: Root account was

2.2.1 Administrative Accounts#

We use ssh key to login, no password managment.

2.2.2 System Accounts#

sudo usermod -L daemon sudo usermod -L bin sudo usermod -L sys sudo usermod -L uucp sudo usermod -L lp sudo usermod -L nobody sudo usermod -L games sudo usermod -L news sudo usermod -L www-data sudo usermod -L mail

2.3 System Account Management Improvement#

Not apply, we have and hub account exist.

2.4 Privileged Account Controls#

Not apply.

3 Password Policy#

Not Apply, We use ssh key to login, no password managment.

4 System Logging Policy#

if the client do not need to centralize the log, skip this step.

  • Edit /etc/rsyslog.conf file
  • Add *.info @syslog.corp.mtrc.com # different client may use different syslog server, make sure change it.
  • sudo systemctl restart rsyslog.service

5 Clock Synchronization#

If the client don't have ntp server, skip this step or just install ntp servie is fine.

  • sudo apt-get install ntp

  • sudo vim /etc/ntp.conf, remark all pool and restrict start lines, add following lines to the end of file. Please take care of the server for different clients.

    server ntp1.corp.mtrc.com prefer server ntp2.corp.mtrc.com server ntp3.corp.mtrc.com server ntp4.corp.mtrc.com restrict default ignore

  • sudo systemctl restart ntp

6 Minimize inetd network services#

We don't have the listed servcies installed.

7 Disable un-necessary services#

We don't have the listed servcies installed. If there is servie un-necessary. run systemctl disable <service name> to disable it.

8 Restrict File/ Directory Permissions to Authorized Users#

Don't have this kind of files.

9 Restrict System Access, Authentication, and Authorization#

9.1 Enable Hidden Passwords#

It use shadow passwords by default, and we do't use password.

9.2 Restrict at/cron to authorized users#

sudo vim /etc/cron.d/cron.allow Add one user per line. If there are no users in /etc/cron.d/cron.allow, only root user will be allowed to use crontab. It's an empty file.

9.3 Restrict crontab file permissions#

We don't have crotnab file

9.4 Set retry limit for account lockout#

It seems this is not suitable for account.

9.5 Session Timeout Policy#

  • edit /etc/profile
  • Add export TMOUT=1800

10 Secure User Accounts and Environment#

10.1 Verify that there are no accounts with empty password field#

We use ssh key to login, no password.

10.2 Secure user home directories#

sudo chmod 750 autotunnel sudo chmod 750 sudo chmod 750 hub

10.3 No user dot-files should be group/world writable#

sudo chmod g-w .* sudo chmod o-w .*

10.4 Set default umask for users#

  • edit /etc/profile
  • Add umask 002, or maybe 027. MTR requested 002.

10.5 Set "mesg n" as default for all users#

  • edit /etc/profile
  • Add mesg n

11 Warning Banners#

  • sudo vim /etc/pam.d/sshd
  • remark this line. # session optional pam_motd.so motd=/run/motd.dynamic
  • sudo systemctl stop motd-news.timer
  • sudo systemctl disable motd-news.timer

12 SNMP Setting#

we don't have SNMP installed.

13 Security Patches#

sudo apt-get update to install all updates.
